NCUI Component Library

NHN Cloud · 2022 — Present

NHN Cloud 콘솔 전반에 사용되는 UI 컴포넌트 라이브러리입니다. 약 4년간 개발·운영하며 전체 57개 코어 컴포넌트 중 9개를 직접 설계·구현했고 56개에 유지보수로 직접 관여했습니다.

  • DateTimePicker 3단계 진화 주도 — 2023년 TimePickerSpinBox·DatePicker·DateRangePicker 등을 신규 개발해 날짜·시각 선택 컴포넌트군을 추가했습니다. 사용 사례 확장에 따라 두 차례 리팩터링을 수행했습니다.
    • 1차: DatePicker 상태 관리 hook과 DateTimeInput 분리, 폴더 구조 정리로 유지보수 용이성 확보
    • 2차: segment(yyyy/mm/dd/hh/mm/AM·PM) 단위 상태를 관리하는 useSegmentManager를 최상위로 승격해 단일 상태 모델로 통합, navigationDate 개념으로 "달력 표시 월"과 "선택 값" 분리
  • Table 컴포넌트 장기 진화 — 초기 children 합성 패턴 도입(2023)부터 가로 스크롤·indeterminate Header Checkbox·이벤트 핸들러 중단 가능 API 등 4년에 걸쳐 점진적으로 확장했고, 디자인 토큰·CSS Modules 마이그레이션을 주도했습니다.
  • LiveEditor 자체 발의·개발 — "정적 코드 블록만으로는 컴포넌트 사용법 학습이 어렵다"는 사용자 의견에서 출발해 공식 업무 없이 직접 발의했습니다. esbuild-wasm 기반 브라우저 번들링, NPM Registry 연동 모듈, 모듈 리졸버 서브경로 해석, 의존성 캐시 전략까지 구현해 임의 패키지를 브라우저에서 즉시 실행하는 환경을 구축했고, 이후 팀 공통 문서 인프라로 자리잡았습니다.

NCUI DataGrid

NHN Cloud · 2024 — Present

스프레드시트 수준의 데이터 편집·조작이 필요한 콘솔 화면을 위해 별도 패키지로 발의·구축한 고급 데이터 그리드입니다. 프로젝트 환경 설정 첫 커밋부터 직접 시작해 약 11개월간 신규 개발을 주도했고, 팀 공동 유지보수 단계에서도 메인 컨트리뷰터로 참여 중입니다.

  • 아키텍처 설계 — TanStack Table을 코어 엔진으로 채택하되, 공개 API와 디자인은 NCUI 컨벤션으로 추상화하는 래퍼 계층을 직접 설계했습니다. 자체 상태 관리에서 컨텍스트 기반 구조로 재구성했습니다.
  • 핵심 기능 구현 — 컬럼 리사이징/DnD/고정/그룹 헤더, Cell/Row/Column 셀렉션, 컨텍스트 메뉴, 복사/붙여넣기, 빌트인 에디터(텍스트·Select·Combobox·Date)와 편집 중 유효성 검사, Undo/Redo 히스토리(편집·setValue·setRow·appendRow 전 동작 대응), Row Expand, Tree Grid, 페이지네이션·정렬·필터·DnD 전 영역의 onBeforeXxx 이벤트, 클립보드 직렬화, 다국어(i18n) 등을 직접 설계·구현했습니다.
  • API & UX 개선 — 외부에서 그리드 인스턴스를 제어하는 데이터 조작·체크 행 조회·편집 종료/취소·셀 선택 API를 일관된 형태로 정비했습니다. 외부 클릭 시 편집 종료, enableColumnSelection·DnD 디커플링 등 정합성 이슈를 지속적으로 보완했습니다.
  • 외부 엔진을 활용하면서도 디자인 시스템과 통합된 일관 API를 제공하는 추상화 레이어 설계, 편집·히스토리·이벤트가 얽힌 복잡한 상태 머신 안정화, 팀원 기여를 위한 폴더·이벤트·API 컨벤션 정립까지 장기 유지보수 친화적 설계 역량을 길렀습니다.

TOAST UI Grid

NHN · TOAST UI팀 · 2021 — 2022

오픈소스 데이터 그리드 컴포넌트를 메인테이닝하면서 대량 데이터 처리, 셀 병합, 선택/편집, 붙여넣기, 행 높이, 포커스 처리 등 복잡한 데이터 UI 기능 개선과 버그 수정을 수행했습니다.

  • 성능 병목 분석으로 100만 셀 데이터 설정 시간을 약 10분→0.6초 수준으로 단축 — 불필요한 전체 데이터 처리와 반복 갱신을 제거했습니다.
  • 동일 값 연속 셀 동적 병합 기능 개발 — 필터·정렬·Drag & Drop·페이지네이션·셀 편집 등 데이터 상태 변화에도 병합 일관성이 유지되도록 처리했습니다.
  • Grid가 단순 테이블이 아니라 대량 데이터·사용자 조작·렌더링 상태가 밀접하게 연결된 애플리케이션이라는 점을 체감했습니다. 복잡한 데이터 UI 개선 시 데이터 변경 흐름과 사용자 조작 시나리오 전체를 함께 고려해야 한다는 점을 배웠습니다.

Console Builder

NHN Cloud · 2025.11 — Present

NCUI 컴포넌트로 콘솔 화면을 비주얼하게 조립하고 코드를 산출하는 노코드형 콘솔 화면 빌더입니다. 2024년 초기 PoC에서 받은 피드백을 기반으로, 2025년 11월부터 모노레포 전환 및 JSXN 직렬화 시스템과 빌더 UI 전반을 새로 설계·구축했습니다.

  • JSXN 직렬화 시스템 설계 — JSON 형태로 표현되는 자체 트리 모델을 설계해 React 코드 ↔ JSXN 양방향 변환을 구현. 초기 jsxn-converter에서 한 번의 구조 개편을 거쳐 jsxn-tree 모델로 전환, 트리 조작과 테스트 가능성을 끌어올렸습니다.
  • 비주얼 에디터 UI 구축 — Canvas, TreeView 패널, 컴포넌트 목록 패널, Property 컨트롤 패널을 직접 설계·연결했습니다. Canvas 다중 선택, TreeView ↔ Canvas 양방향 호버·선택 동기화(자동 펼침/스크롤), 외부 드롭, alt+Click 부모 선택, 일괄 삭제, 그룹화/해제, 페이지 간 노드 복사/붙여넣기 등 실제 편집 도구 수준의 인터랙션을 구현했습니다.
  • 템플릿·패턴 시스템 — 컴포넌트를 묶어 재사용 가능한 단위로 만드는 템플릿(중첩 참조 재귀 해석, unlinkTemplate, 편집 진입/돌아가기 흐름)과 FormField·FormLayout 같은 도메인 단위 합성 블록을 제공하는 패턴 레지스트리를 도입했습니다.
  • Undo/Redo 히스토리, URL 공유로 작업 상태 재현, 캔버스 미리보기 및 Framework 토글 등 빌더의 실사용성을 끌어올렸습니다. 외부 PoC를 인수해 본 제품 수준으로 끌어올린 레거시 정리 능력을 함께 길렀습니다.

협업 방식

Cross-functional · Designer · Devs

오픈소스 메인테이너로 일하던 시절부터 외부 사용자 문의를 직접 응대해 왔고, 사내 라이브러리 개발자로 옮겨온 뒤에는 디자이너·다른 팀 개발자와 더 긴밀하게 호흡을 맞추고 있습니다.

  • 디자이너와의 기술적 소통 — 인라인/블록 차이, 폰트 메트릭, 베이스라인 정렬처럼 디자이너가 직관적으로 인지하기 어려운 렌더링 변수를 "왜 이렇게 보일 수밖에 없는가"를 비유와 단계적 설명으로 풀어내고 디자인 의도를 만족시킬 대안까지 제시. 누적된 소통으로 디자이너와 개발자가 서로의 제약을 이해한 상태에서 함께 결정하는 기반을 만들었습니다.
  • 사용자 피드백 루프 운영 — 사내 도구 사용자(다른 팀 개발자)에게 먼저 피드백을 요청하고, 문의 시 단순 응답을 넘어 "왜 그렇게 동작하는지"를 정리해 공유. 같은 질문이 반복되지 않는 구조를 만들고 라이브러리 신뢰의 토대를 다졌습니다.
  • 기능을 만드는 것만큼이나 그 결정을 다른 직군에게 설득력 있게 전달하는 것이 라이브러리 개발자의 핵심 역량이라는 점을 체감하며, 어려운 기술 개념을 비전공자에게 풀어내는 커뮤니케이션 능력을 함께 길렀습니다.
